## Email mời họp (1 mẫu)

### Rule 18 — Template email xin reschedule

*JP: リスケ依頼メール・テンプレート*

```
件名：【リスケ依頼】〇月〇日(〇) 〇〇:〇〇 〇〇会議

〇〇様
（CC：〇〇様）

お世話になっております。
ティエンファットの〇〇でございます。

直前のご連絡となり恐縮ですが、〇〇のため、当初予定の〇月〇日(〇) 〇〇:〇〇〜〇〇:〇〇 の会議に参加できなくなりました。
誠に申し訳ございません。

下記の通り代替候補を3つご提案させていただきます。ご都合の良い候補をご返信いただけますと幸いです。

──────────────────────
■ 代替候補
  ① 〇月〇日(〇) 〇〇:〇〇〜〇〇:〇〇 (JST)
  ② 〇月〇日(〇) 〇〇:〇〇〜〇〇:〇〇 (JST)
  ③ 〇月〇日(〇) 〇〇:〇〇〜〇〇:〇〇 (JST)

■ 形式：〇〇（リンク：https://...）

会議室・Zoom リンクは変更不要です。

何卒よろしくお願い申し上げます。
```

📝 **Ghi chú VN:** Tiêu đề ghi rõ '[Xin reschedule]'. Lý do nói 1 dòng ngắn (không cần chi tiết nội bộ). Tối thiểu 3 option, bắt buộc kèm JST. Show rõ trách nhiệm update lịch để khách không phải lo.
📝 **Ghi chú JP:** 件名で「リスケ依頼」と明記。理由は1行で簡潔に（社内詳細は不要）。候補は最低3つ、JST 表記必須。更新責任を明示することで相手の負担を取り除く。


## Checklist (3 mẫu)

### Rule 12 — Checklist phòng ngừa báo cáo sót — 10 mục

*JP: 報告漏れ防止 セルフチェック10項目*

```
【毎日17時／毎週金曜17時に実施】

□ 1. 本日受信したメール／Slack DM はすべて確認したか？
□ 2. 本日返信が必要なものに返信したか？（未返信件数：〇件）
□ 3. 口頭で受けた依頼を書面（メール／Slack）に変換したか？
□ 4. 約束した期日（社内・社外）はすべて期日内に完了したか？
□ 5. 期日内完了が難しいものは事前にリスケ依頼したか？
□ 6. 上司への daily 進捗報告を送ったか？（DDBN形式）
□ 7. 顧客への進捗共有が必要なものは送ったか？
□ 8. 悪い知らせは発生から30分以内に第一報を入れたか？
□ 9. 検収・承認・OK は書面で取得したか？（口頭OKは書面化済か）
□ 10. 来週／翌日の予定で先方に共有すべき変更はないか？

【記録欄】
  - 未返信件数：〇件（具体内容：〇〇）
  - 期日達成率：〇/〇 件
  - 口頭→書面変換件数：〇件
  - 翌日／翌週への持ち越しタスク：〇件
```

**Bản Việt placeholder:**

```

[ ] 1. Đã đọc hết mail / Slack DM nhận hôm nay chưa?
[ ] 2. Đã reply những cái cần reply chưa? (Số chưa reply: 〇)
[ ] 3. Việc nhận bằng miệng đã chuyển thành văn bản (mail/Slack) chưa?
[ ] 4. Deadline đã hứa (nội bộ + ngoại bộ) đã xong đúng hạn chưa?

[ ] 6. Đã gửi báo cáo tiến độ daily cho sếp chưa? (Format DDBN)

[Mục ghi chép]
  - Số chưa reply: 〇 (nội dung: 〇〇)
  - Tỉ lệ đạt deadline: 〇/〇
  - Số việc verbal đã convert thành văn bản: 〇
  - Task chuyển sang ngày/tuần sau: 〇
```

📝 **Ghi chú VN:** Đặt alarm 17h hàng ngày để bắt buộc chạy. Mục đích không phải tick hết 10 mục mà là visualize cái nào còn pending. Báo 0 hết = đáng nghi.
📝 **Ghi chú JP:** 毎日17時にアラームで実行を強制。10項目を全部「チェック」より、未対応がある項目を可視化することが目的。0件報告は逆に怪しい。

### Rule 38 — Checklist Self-diagnostic Hou-Ren-Sou 15 mục

*JP: Hou-Ren-Sou 自己診断 15項目チェックリスト*

```
【月末（毎月最終金曜EOD）に実施】
評価: A=完全実施 / B=概ね実施 / C=一部不足 / D=未実施
判定: 必ず Slack履歴 + Decision Register + 客先メール の3点と照合

──────────────────────────────────────────
■ 報告 (Hōkoku) — 5項目
  □ ① 結論先出し: 報告冒頭で『結論:〇〇』と明示できているか?
  □ ② Fact / Opinion 分離: 『事実として〜』『私見では〜』を意識して使っているか?
  □ ③ タイムリー: 悪い知らせを発生から30分以内に第一報したか? (Sev1: 5分)
  □ ④ 期限明示: Next action に必ず deadline が付いているか?
  □ ⑤ 4文構造: 結論 → 事実 → 私見 → 対応 の構造で報告したか?

■ 連絡 (Renraku) — 5項目
  □ ⑥ 即時性: 客先からの連絡を24h以内に少なくとも ack しているか?
  □ ⑦ 客先 tone: クレーム時に defensive せず、傾聴+復唱できたか?
  □ ⑧ Channel 選択: Slack / Notion / Email / Outlook を題材ごとに使い分けたか?
  □ ⑨ Audience 選択: @here / @channel / DM を適切に使い分けたか?
  □ ⑩ EOD ログ: WFH 日に EOD ログを Slack に投稿したか?

■ 相談 (Sōdan) — 5項目
  □ ⑪ 複数案持参: 上司・客先相談時に最低2-3案 + pros/cons を持参したか?
  □ ⑫ 仮説提示: 自分の仮説と理由を1〜2行で提示したか?
  □ ⑬ 判断材料を求める: 『答えではなく見落とし観点』を聞く姿勢だったか?
  □ ⑭ DR化: 相談で決まったことを Decision Register に1行残したか?
  □ ⑮ 客先敬語: 客への相談で『ご相談させていただけますでしょうか』を使えたか?

──────────────────────────────────────────
【記録欄】
  - 評価分布: A〇 / B〇 / C〇 / D〇
  - C-D 評価の具体ケース:
    1. 〇〇
    2. 〇〇
  - Action Plan (5月末再診断 target):
    1. 〇〇
    2. 〇〇
  - 再診断予定日: 〇月〇日
  - DR エントリ番号: DR-YYYY-MMDD
```

**Bản Việt placeholder:**

```
[Chạy cuối tháng (thứ Sáu cuối cùng EOD)]
Đánh giá: A = đầy đủ / B = cơ bản / C = thiếu một phần / D = chưa làm
Xác định: BẮT BUỘC đối chiếu Slack history + Decision Register + mail khách

──────────────────────────────────────────
[Báo cáo (Hōkoku) — 5 mục]
  [ ] [1] Kết luận trước: Mở báo cáo có 'Kết luận: 〇〇' rõ không?
  [ ] [2] Tách Fact/Opinion: Có ý thức dùng 'Sự thật là...' và 'Theo ý em...' không?
  [ ] [3] Kịp thời: Tin xấu báo cáo đầu trong 30 phút từ phát sinh? (Sev1: 5 phút)
  [ ] [4] Show deadline: Next action có deadline cụ thể không?
  [ ] [5] Cấu trúc 4 câu: Kết luận → Sự thật → Ý kiến → Cách xử lý đã làm chưa?

[Liên lạc (Renraku) — 5 mục]
  [ ] [6] Tức thì: Liên lạc khách trong 24h ít nhất ack chưa?
  [ ] [7] Tone với khách: Lúc khiếu nại có lắng nghe + lặp lại thay vì defensive không?
  [ ] [8] Chọn channel: Có chia Slack/Notion/Email/Outlook theo từng việc không?
  [ ] [9] Chọn audience: @here / @channel / DM đã dùng đúng chưa?
  [ ] [10] EOD log: Ngày WFH có post EOD log lên Slack chưa?

[Tham vấn (Sōdan) — 5 mục]
  [ ] [11] Mang nhiều phương án: Tham vấn sếp/khách có mang 2-3 option + pros/cons không?
  [ ] [12] Giả thuyết: Có trình bày giả thuyết + lý do 1-2 dòng không?
  [ ] [13] Hỏi dữ kiện: Có thái độ hỏi 'axis bỏ sót' thay vì 'cái nào hơn' không?
  [ ] [14] DR hóa: Việc đã quyết có lưu 1 dòng vào Decision Register không?
  [ ] [15] Kính ngữ với khách: Tham vấn khách có dùng 'ご相談させていただけますでしょうか' không?

──────────────────────────────────────────
[Mục ghi chép]
  - Phân bổ đánh giá: A〇 / B〇 / C〇 / D〇
  - Case cụ thể của C-D:
    1. 〇〇
    2. 〇〇
  - Action Plan (target lần chấm sau cuối tháng 5):
    1. 〇〇
    2. 〇〇
  - Ngày diagnostic lại: 〇/〇
  - Số DR entry: DR-YYYY-MMDD
```

📝 **Ghi chú VN:** Chấm khắt khe mới đúng. A ≥12 = tự động cảnh báo self-bias. C-D ≥ 3 thì set 1-on-1 với mentor ngay. Mỗi 3 tháng vẽ chart xu hướng để thấy tăng trưởng dài hạn.
📝 **Ghi chú JP:** 評価は厳しめが正解。A 12個以上は自動的に self-bias 警告。C-D が 3項目以上なら mentor 1-on-1 を即座にセット。3ヶ月毎に推移をグラフ化して長期成長を見える化する。

### Rule 02 — Template báo cáo tiến độ Daily (Slack)

*JP: Daily 進捗報告（Slack）テンプレート*

```
【Daily 〇月〇日】

  - 〇〇（完了）
  - 〇〇（〇/〇 対応）
Doing:
  - 〇〇（進捗 〇%）

  - なし／〇〇（〇〇待ち）
Next (今日):
  - 〇〇までに〇〇着手
  - 〇〇までに〇〇完了見込み
```

**Bản Việt placeholder:**

```
[Daily 〇/〇]
Done:
  - 〇〇 (xong)
  - 〇〇 (xử 〇/〇)
Doing:
  - 〇〇 (tiến độ 〇%)

  - không / 〇〇 (chờ 〇〇)

  - Bắt tay 〇〇 trước 〇〇
```

📝 **Ghi chú JP:** Slack の場合は1行ずつ改行。スレッド化禁止（流し読み出来なくなる）。Blocked が「なし」でも欄を残す。
