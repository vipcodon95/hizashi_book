# Rule 21 — 関係のない多人数への一斉送信は必ずBCC

> **要点.** **お互いに面識のない複数の相手に同じメールを送る**場面 — 顧客リスト、取引先リスト、複数部署への通知 — では**BCCが必須**。`To:` または `Cc:` に並べると、**各人のメールアドレスが他の全員に公開**されてしまう。プライバシー侵害であり、場合によっては個人情報保護法（改正個人情報保護法令和4年施行）違反になる。
>
> キーワード：**「一斉送信＝BCC」**。

## 背景

年末、ズンは弊社の50社の顧客に年末の挨拶メールを送る必要がある。これらの顧客同士は競合または取引先の関係 — 誰もがティエンファットの顧客だと**互いに知るべきではない**。

## 悪い例 — 全員を `To:` に入れる

> **To:** matsumoto.akira@hakuo.co.jp, hoa.tanaka@vpc.co.jp, sato.tetsuo@phuongnam-j.co.jp, ... (残り47件)
>
> 各位、
>
> 年末にあたり、弊社より...

**結果.**
- 松本は自分のメールを開いた瞬間、49社の他顧客リストを目にする → ティエンファットの顧客リスト全体を把握。
- 誰か1人が **「全員返信」** を誤って押すと、49人に対してスパム的に届く。
- 白鷗の競合が50社リスト内にいると、白鷗がティエンファットを利用していることが競合にバレる。
- 誰かのメールがハッキングされれば、スパマーが50件分の送信先リストを手に入れる。

**法的観点.** 日本の改正個人情報保護法（令和4年4月施行）および令和5年のプライバシー強化指針において、メールアドレスは個人情報。業務必要性のない第三者への開示は、場合によっては行政処分対象。

## 良い例 — `Bcc:` に入れる

> **To:** info@tienphat.com.vn *（自社の代表アドレス）*
> **Bcc:** matsumoto.akira@hakuo.co.jp, hoa.tanaka@vpc.co.jp, sato.tetsuo@phuongnam-j.co.jp, ... (残り47件)
>
> 各位、
>
> 年末にあたり、弊社より...

**起こること：**
- 各顧客は自分だけにメールが届いた形で受信（`To:` には `info@tienphat.com.vn` のみ）。
- 他のBCC先の存在を知ることはない。
- 誰かが「全員返信」を押しても、ティエンファットにしか返らない — 他48人に影響なし。

## To / Cc / Bcc 使い分けの指針

| メールの種類 | 使うもの |
|-------------|---------|
| 1名への直接送信 | その人を `To:` |
| 1名宛、関係者1〜3名が知るべき | 主受信者を `To:`、関係者を `Cc:`（紹介付き — Rule 20） |
| 小人数グループ（3〜7名）、お互い顔見知り | 全員を `To:` — 既に相互認識あり |
| 10名以上、**相互認識あり**（例：部署全員） | 全員を `To:` または部署エイリアス（`sales@tienphat.com.vn`） |
| 10名以上、**相互認識なし** | **`Bcc:` 必須** |
| 顧客への一斉通知 | **`Bcc:` 必須** |

## BCCが不適切な場合

### ケース1 — 上司を「仕掛けとして」BCC

取引先にメールを送る際、取引先に**告げずに**上司をBCCで追えさせる。上司が誤って「全員返信」を押すと → 取引先が気づく → 信頼失墜。組織文化の透明性の欠如も表す。

**正しい方法：**
- 送信後に上司へ転送。
- あるいはCCして紹介する（「フオン副部長をご参考までにCCしております」） — 公式化する。

### ケース2 — スレッドで新しい受信者を追加

スレッドが話題を変え、新たに誰かを参加させる必要がある時、BCCで「こっそり」入れない。新しいメール本文で明示する：「トゥアンリーダーをここから参加させます、継続してご確認いただきます。」

### ケース3 — 「説明したくないから」BCC

*この人をなぜ加えるか説明したくない*からBCC、なら自問：「説明すると何か問題があるか？」 問題があるなら、CCもBCCもしない。問題なければ、CCして紹介する。

## 一斉送信時のBCC設定の安全手順

1. **メール本文を先に書く。** まだ宛先を入れない。
2. **`To:` に自分、または会社代表アドレス**（`info@tienphat.com.vn`）を入れる。
3. 顧客メールリストをコピー、**`Bcc:` に貼り付け**、`To:` や `Cc:` に混入していないか2回確認。
4. **再度確認する。** 一度誤送信すると取り消せない。
5. リストが100件を超える場合、BCCはスパムフィルタで弾かれやすくなる — 専用のメールマーケティングツール（Mailchimp、SendGrid、配配メール）を使用すべき。

## 使えるフレーズ

> **「相互面識のない複数人＝BCC、Cc ではない。送信前に2回確認。」**

*使うとき：* メールに5以上の宛先があり、互いに面識があるか確信できない時。迷ったら**デフォルトはBCC**。
