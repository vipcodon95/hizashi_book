# Rule 22 — 本文か添付か — 3つの判断基準

> **要点.** 本文と添付の境目は、やや長めの内容を書くたびに出てくる問い。絶対のルールはないが、**3つの基準**で素早く判断できる：(1) **長さ**、(2) **読み方**（ざっと見るか、精読するか）、(3) **保管性・検索性**。誤判断は「添付したのに誰も開かない」「本文が長すぎて読まれない」のどちらかに至る。
>
> キーワード：**「本文か添付かの3基準」**。
>
> **関連：** Rule 65（添付ファイル名を本文で明示）、Rule 66（各添付に1行要約）、Rule 67（容量10MB制限）。

## 3つの基準

### 基準1 — 長さ

| 内容の長さ | 推奨場所 |
|----------|---------|
| 5行以下 | 本文 |
| 5〜15行、構造明確 | 本文（箇条書き、見出しで） |
| 15〜30行 | 相手が早く判断する必要があれば本文、参考資料なら添付 |
| 30行超 | 添付、本文は要約のみ |

**理由：** スマートフォンで本文30行超はスクロールを強いる — 読み疲れる。しかし添付は「もう1手間」（ダウンロードと展開）を要するので、短い内容をファイル化するのも失礼。

### 基準2 — 読み方

核心の問い：*相手は一度ざっと読むだけか、それとも何度も参照するか？*

| 読み方 | 推奨場所 |
|-------|---------|
| 1回ざっと把握、後で戻らない | 本文 |
| 後で何度も参照する | 添付（保管しやすい） |
| 表データ、数値をコピーする | Excelファイル |
| 画像・図・組織図 | 添付（画像またはPDF） |
| 文字のみ、複雑な書式なし | 本文 |

### 基準3 — 保管・検索性

**正式記録として保存する**必要があるもの（契約書、議事録、公式報告書）は必ずファイル化。本文は体系的な保管が難しい — メール本文を Google Drive フォルダに入れて検索、はできない。

| 種類 | 推奨場所 |
|------|---------|
| 契約書、付属書 | PDF（電子署名付きなら尚良） |
| 公式見積書 | PDF |
| 議事録、決議書 | Word/PDF |
| 数値データ | Excel |
| 議論中のドラフト | 本文（インラインコメント可能） |
| 進捗状況アップデート | 本文 |
| 仕様書、デザイン文書 | 添付（PDF/Word）、本文で要約 |

## 例 — 同じ内容、2通りの整理

### 状況

ズンがフオン副部長に週報を送る。週報内容：15行のテキスト、10列×8行の数値表、進捗グラフ2点。

### 悪い例1 — 全て本文に詰め込む

ズンが10列の表を本文にペースト → スマホで表示崩れ、読めない。グラフは本文（テキスト）に挿入できない → ズンは文字で説明 → 視覚情報の喪失。

結果：フオンはスマホで報告書を読めず、PCに戻るまで読まない。24時間ゴールデンウィンドウ（Rule 04）を逃す。

### 悪い例2 — 全て添付、本文1行

> フオン副部長、
>
> 第19週の報告書を添付いたします。
>
> ズン

フオンは本文1行＋ファイルを見る。今忙しい、ファイルを開く時間がない、「後で読む」 — その「後で」は来ない。

### 良い例 — 本文に要約、添付に詳細

> フオン副部長、
>
> お疲れ様です、BDのズンです。第19週の週報をお送りいたします。ご確認いただきたい3点：
>
> - **項目A**（フロントエンド）：順調、金曜にデモ予定。
> - **項目B**（白鷗API）：要件確認待ちで2日遅延、対応中。
> - **項目C**（予算）：追加50万円、5/15までにご承認必要。
>
> 数値詳細と進捗グラフは添付ファイル（`weekly_report_w19.xlsx`、シート2枚）をご参照ください。
>
> よろしくお願いいたします。
> ズン

- フオンはスマホで30秒読めば大枠を把握。
- 数値が必要ならPCで後でファイルを開く。
- **ファイル内に埋もれる情報**は発生しない。

## 鉄則 — 「ファイルには必ず本文要約を付ける」

**添付ファイルを本文要約なしで送ることは絶対にしない。** 最低3文：

> 「[ファイル名]をお送りいたします。[ファイル内容を1文で]。[ご注目いただきたい最重要ポイント]。」

要約なしでファイルだけ送るのは、受信者に*ファイル名から内容を推測させる*行為 — 最悪のUX。

## 使えるフレーズ

> **「短く→本文。長く、保管性、表があるなら→添付。添付には必ず本文3文の要約を付ける。」**

*使うとき：* 送信ボタンを押す前、メールに添付がある場合。本文がファイルを開かなくても**行動できる**情報量を持っているか、ファイルは必要な深掘り用に用意されているかを確認する。
