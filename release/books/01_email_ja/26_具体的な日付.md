# Rule 26 — 「明日」「来週」は罠 — 具体的な日付を書く
> **要点.** 「明日、5月15日（木）10:00」と書く。**「明日」だけは書かない**。水曜23:00に送ったメール — 「明日」は木曜？ それとも日付が変わって金曜？ 海外宛なら時差はどう？ 必ず：曜日＋日付＋時刻＋（必要なら時差）を明記。
>
> キーワード：**「絶対日時」**。

## 悪い例
> 「明日までに報告書を提出してよろしいでしょうか？」
> 「来週金曜日の午後にお会いしましょう。」

**問題点：**
- 「明日」は送信時？ 受信時？ 23:50に送ると大きく違う。
- 「来週金曜」— 今週末の直後の金曜？ それともその次？ 解釈が分かれる。
- 「午後」— 13:00? 17:00?

## 良い例
> 「**5月15日（木）17:00までに**報告書を提出してよろしいでしょうか？」
> 「**5月23日（金）14:00（JST）**にお会いしましょう。」

## 推奨フォーマット
| 種類 | フォーマット |
|------|------------|
| 日付＋曜日 | 5月20日（火） |
| 時刻付 | 5月20日（火）14:00 |
| 時差付（海外宛） | 5月20日（火）14:00（JST） |
| 日没締切 | 5月20日（火）23:59まで |
| 日初締切 | 5月21日（水）09:00まで |
| 時間帯 | 5月20日（火）14:00–15:30 |

## よくある罠
- **「週明け」** — 月曜？ 火曜？ 水曜？
- **「月末」** — 月の28日？ 31日？ 最終営業日？
- **「なるべく早く」** — 1時間以内？ 1日？ 1週間？

数値で書かないと、10人に10通りに解釈される。

## 使えるフレーズ
> **「曜日＋日付＋時刻＋（時差）。4項目、全て埋める。」**
