# Rule 17 — 署名は2種類を使い分ける：顧客向けのフル版と社内向けの短縮版
> **要点.** 現代のメールクライアント（Gmail、Outlook）は**複数の署名を登録**し、作成時に選択できる。少なくとも **2種類** 必要：(A) *フル版* — 顧客・取引先・初対面向け。(B) *短縮版* — 社内向け、短い名前と部署のみ。フル版を社内の親しい同僚に使うのは大げさ、短縮版を顧客に使うのは非プロ。
>
> キーワード：**「2種類の署名」**。

## バージョンA — フル版（顧客、取引先、初対面向け）
```
何卒よろしくお願い申し上げます。

ズン (Zung Tran)
営業部 BD担当
ティエンファット社
─────────────────
TEL: +84-90-xxxx-xxxx
Email: zung.tran@tienphat.com.vn
Web: https://www.tienphat.com.vn
ホーチミン市1区グエンフエ通りxx ビル 5F
```

**使用場面：**
- 顧客（松本PMなど）への送信。
- 取引先・仕入先への初回連絡。
- 見積書・契約書・提案書送付。
- 社を代表して外部に発信するメール全般。

**省略すべきでない要素：**
- フルネーム。
- 役職 — 相手がどのレベルで対話すべきか判断するため。
- 会社名 — 正式な法人名。
- 電話番号 — メールが届かない時のバックアップ連絡手段。
- メールアドレス — 差出人にも入っているが、コピー操作のために再掲。

**検討する要素：**
- オフィス住所 — 会社が複数拠点を持つ場合、どの拠点かを明示。
- 会社ロゴ — マーケ部が公式画像（カラー、サイズ）を提供している場合のみ。ネット拾いは絶対NG — スマホで崩れる。
- スローガン・タグライン — 社風による。社として統一している場合は使う。自分勝手に付けない。

## バージョンB — 短縮版（社内、親しい同僚向け）
```
よろしくお願いいたします。
ズン（BD）
```

もしくは更に簡潔に：

```
ズン
```

**使用場面：**
- 社内メール（ティエンファット内）。
- 同日に継続中のスレッド、同僚間。
- 短い返信や、頻繁にやり取りする相手。

**なぜバージョンBが必要か：**
- フオン副部長に週40通送信するたびフル署名（ロゴ、住所、スローガン付き）— *大げさ*、毎通が名刺のよう。
- 一部の会社は社内メール容量制限あり。ロゴ付き署名で受信箱が早く埋まる。
- 20通以上のスレッドでは、短縮版の方が視覚的に軽い。

## 悪い例 — 誤バージョン使用
### ケース1：フル版を親しい同僚に使う
ズンがフオンに「承知しました、本日中に対応いたします。」の一言の返信に、ロゴ・住所・スローガン付き8行のフル署名を付ける。

→ フオンにとっては、同僚の返信ではなく**マーケメール**を受け取った感覚。

### ケース2：短縮版を顧客に使う
ズンが松本（白鷗）に重要な見積書を送るメールに「ズン」だけで署名。

→ 松本はズンの電話番号を知らず、所属会社も役職も分からない。緊急時に連絡するには古いメールを探すしかない。「この人はプロではない」という印象が既に定着。

## 設定方法
### Gmail
設定 → すべての設定を表示 → 全般 → 署名 → 新規作成 → 「フル」「短縮」と名付け → 新規メールと返信で別デフォルトを設定可能。

### Outlook（デスクトップ）
ファイル → オプション → メール → 署名 → 新規 → 2種類作成 → アカウント別に割り当て。

### Outlook（Mac）
Outlook → 環境設定 → 署名 → 新規追加。

メール作成時、Compose画面上部の**署名選択**ドロップダウンで1秒切替可能。

## バージョンC — オプション（英語版、海外取引向け）
海外パートナーとやり取りするなら、バージョンCを英語で作成：

```
Best regards,

Zung Tran
Business Development Executive
Tien Phat Co., Ltd.
```

フル版よりやや簡潔、英語圏の短いスタイル。

## 使えるフレーズ
> **「2種類の署名。送信前に切り替える。1秒で完了。」**

*使うとき：* 今週月曜に2種類設定する。その後、あなたが送るすべてのメールは適切な文脈を反映する — 顧客にはプロフェッショナル、同僚には軽やかに。
