# Rule 18 — 繰り返し送るメールは個人テンプレート化する
> **要点.** 同じ型のメールを **3回以上** 書いているなら — 週報、日程変更依頼、請求書リマインド、提案、断り — 毎回ゼロから書くのは時間の無駄。Draftsもしくは Templates機能に**個人テンプレート**を保存する。以降は複製→3箇所埋める→送信の作業だけ。
>
> キーワード：**「3回以上＝テンプレ化」**。

## 背景
ズンは毎週月曜にフオン副部長へ週報を送っている。毎回ゼロから書いて20〜30分かけている。

## テンプレート化しない場合の実費
あなたが1週間に書く量：
- 週報1通（20分）
- 日程変更依頼2通（5分×2）
- 請求書リマインド3通（5分×3）
- 新規提案2通（15分×2）

合計：**約1時間15分/週**、構造を書き直すだけの作業。年間約**65時間**、メールの型を書くために**8営業日**消費。

## 週報テンプレートの例
Draftsに `[TEMPLATE] 週報` という名前で保存：

```
件名：【ビンミン】第__週報告 — 副部長にご確認いただきたい__点

フオン副部長、

お疲れ様です、BDのズンです。

第__週のビンミンプロジェクト週報をお送りいたします。ご確認いただきたい__点：

1. [項目1] — [状態：順調/遅延/要注意]
   詳細: ...

2. [項目2] — [状態]
   詳細: ...

3. [項目3] — [状態]
   詳細: ...

今週ご判断いただきたい事項：
- [判断事項1 — 期限]
- [判断事項2 — 期限]

よろしくお願い申し上げます。
ズン
```

毎週月曜、ズンの作業は：
1. このDraftを複製。
2. 週番号を入れる。
3. 3項目と状態を入れる。
4. 要承認事項を入れる。
5. 送信。

所要時間：**5〜7分**（以前の20〜30分に対し）。

## テンプレート化しておくべき代表例
| テンプレート | 頻度 |
|-------------|------|
| 週報 | 毎週 |
| 月報 | 毎月 |
| 日程変更依頼 | 不定期発生時 |
| 請求書期限超過リマインド | 経理から顧客 |
| 新規提案・サービス紹介 | 新規リード対応 |
| 断り（関係維持型） | 参加できない時 |
| 会議後のお礼 | 重要会議の後 |
| 新入社員紹介 | 新人・インターン入社時 |
| 業務引継ぎ | 長期休暇・異動時 |

## テンプレート庫の整理方法
### 方法1 — Drafts でプレフィックス
Draftsに保存、全て `[TEMPLATE]` で始める名前：
- `[TEMPLATE] 週報`
- `[TEMPLATE] 日程変更`
- `[TEMPLATE] 請求書リマインド`

利点：ツール不要、シンプル。
欠点：Draftsに通常の下書きとテンプレが混ざり混乱しやすい。

### 方法2 — Templates機能を使用
- **Gmail：** Settings → Advanced → Templates → Enable。作成画面のMenu（3点）→ Templates → Save draft as template.
- **Outlook（デスクトップ）：** 名前を付けて保存 → *Outlook Template (.oft)* 形式。
- **Outlook Web / Microsoft 365：** *My Templates* アドイン利用。

利点：テンプレが別管理、いつでも挿入可能。
欠点：初期設定に2分必要。

### 方法3 — マシン上のテキストファイル
デスクトップに `mail_templates.md` を作成、全テンプレートを格納。必要時にファイルを開いてコピペ。

利点：メールクライアント非依存（複数受信箱使用時に便利）。
欠点：ウィンドウを1つ余計に管理する必要。

## テンプレートを更新すべきサイン
- ペーストの度に*同じ箇所*を書き換えている → その箇所をプレースホルダ `[...]` にする、もしくはテンプレ本体に含めてしまう。
- *全く使わない*部分がある → 削除。
- 3ヶ月、同じテンプレに対する反応が一様に薄い → 文体が飽きられた、刷新時期。

## 使えるフレーズ
> **「3回以上書いたらテンプレ化。設定5分、年間8営業日節約。」**

*使うとき：* 次にメールを書き始める時、自問する *「このタイプのメールはもう3回書いたか？」*。YESなら今回書き終わったらテンプレ化。次回からは「埋めるだけ」の作業。
